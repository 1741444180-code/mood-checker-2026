## 三、设计要求

### 3.1 风格定位

- **整体风格：** 温暖、治愈、简约
- **目标用户：** 需要情绪记录和追踪的人群
- **设计关键词：** 友好、轻松、可信赖

### 3.2 心情颜色系统

| 心情 | 颜色 | 使用场景 |
|------|------|----------|
| 😄 开心 | #FFD700 金黄 | 积极情绪、高亮 |
| 😊 平静 | #90EE90 浅绿 | 平和状态、成功 |
| 😔 低落 | #87CEEB 天蓝 | 消极情绪、提醒 |
| 😠 生气 | #FF6B6B 红色 | 警告、强烈情绪 |
| 😰 焦虑 | #DDA0DD 紫红 | 关注、需要帮助 |
| 😴 疲惫 | #D3D3D3 灰色 | 中性、未打卡 |
| 🤩 兴奋 | #FF69B4 粉红 | 特殊时刻、庆祝 |

### 3.3 响应式要求

- **PC 端：** 1920x1080（桌面浏览器）
- **平板端：** 768x1024（iPad 等）
- **手机端：** 375x667（iPhone SE 等）
- **触摸友好：** 手机端按钮高度≥44px

### 3.4 组件库

使用 shadcn/ui 作为基础组件库，自定义样式：
- 按钮（主要、次要、幽灵）
- 输入框（文本、密码、多行）
- 卡片（信息卡片、统计卡片）
- 选择器（心情选择、标签选择）
- 日历组件
- 图表组件
