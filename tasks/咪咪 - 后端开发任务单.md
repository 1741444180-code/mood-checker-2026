## 三、数据库表结构

### 3.1 users（用户表）

```prisma
model User {
  id        Int      @id @default(autoincrement())
  username  String   @unique @db.VarChar(50)
  email     String   @unique @db.VarChar(100)
  password  String   @db.VarChar(255)
  avatar    String?
  createdAt DateTime @default(now())
  lastLogin DateTime?
  timezone  String   @default("UTC") @db.VarChar(50)
  
  moodRecords   MoodRecord[]
  moodTypes     MoodType[]
  settings      UserSettings?
  comments      Comment[]
}
```

### 3.2 mood_records（心情记录表）

```prisma
model MoodRecord {
  id        Int      @id @default(autoincrement())
  userId    Int
  date      DateTime @db.Date
  moodLevel Int
  moodType  String
  note      String?  @db.Text
  tags      String?  @db.VarChar(100)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  
  user    User      @relation(fields: [userId], references: [id])
  comments Comment[]
  
  @@unique([userId, date])
  @@index([userId])
  @@index([date])
  @@index([moodType])
}
```

### 3.3 mood_types（心情类型表）

```prisma
model MoodType {
  id        Int      @id @default(autoincrement())
  userId    Int?
  name      String   @db.VarChar(50)
  color     String   @db.VarChar(7)
  icon      String   @db.VarChar(50)
  sortOrder Int      @default(0)
  isSystem  Boolean  @default(false)
  createdAt DateTime @default(now())
  
  user User? @relation(fields: [userId], references: [id])
  
  @@index([userId])
  @@index([isSystem])
}
```

### 3.4 user_settings（用户设置表）

```prisma
model UserSettings {
  id                Int      @id @default(autoincrement())
  userId            Int      @unique
  reminderTime      String?  @db.Time
  reminderEnabled   Boolean  @default(false)
  weeklyReport      Boolean  @default(false)
  publicProfile     Boolean  @default(false)
  theme             String   @default("light") @db.VarChar(50)
  createdAt         DateTime @default(now())
  updatedAt         DateTime @updatedAt
  
  user User @relation(fields: [userId], references: [id])
  
  @@index([userId])
}
```

### 3.5 comments（评论表）

```prisma
model Comment {
  id            Int      @id @default(autoincrement())
  moodRecordId  Int
  userId        Int?
  content       String   @db.Text
  status        String   @default("pending") @db.VarChar(20)
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
  
  moodRecord MoodRecord @relation(fields: [moodRecordId], references: [id])
  user       User?      @relation(fields: [userId], references: [id])
  
  @@index([moodRecordId])
  @@index([userId])
  @@index([status])
}
```

## 四、API 接口设计

### 4.1 用户认证

```typescript
POST /api/auth/register
Body: { username, email, password }
Response: { token, user }

POST /api/auth/login
Body: { email, password }
Response: { token, user }

POST /api/auth/logout
Headers: { Authorization: Bearer <token> }
Response: { success: true }

GET /api/auth/me
Headers: { Authorization: Bearer <token> }
Response: { user }
```

### 4.2 心情记录

```typescript
POST /api/moods
Headers: { Authorization: Bearer <token> }
Body: { date, moodType, moodLevel, note?, tags? }
Response: { record }

GET /api/moods
Headers: { Authorization: Bearer <token> }
Query: { month?, startDate?, endDate? }
Response: { records }

GET /api/moods/:id
Headers: { Authorization: Bearer <token> }
Response: { record }

PUT /api/moods/:id
Headers: { Authorization: Bearer <token> }
Body: { moodType?, moodLevel?, note?, tags? }
Response: { record }

DELETE /api/moods/:id
Headers: { Authorization: Bearer <token> }
Response: { success: true }
```

### 4.3 统计

```typescript
GET /api/stats
Headers: { Authorization: Bearer <token> }
Query: { range } // 7, 30, 90
Response: { 
  totalCount,
  moodDistribution,
  continuousDays,
  averageScore
}

GET /api/stats/trend
Headers: { Authorization: Bearer <token> }
Query: { range }
Response: { trendData }

GET /api/stats/rankings
Query: { type, range }
Response: { rankings }
```
