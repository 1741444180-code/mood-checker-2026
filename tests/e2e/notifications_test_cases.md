# 通知页面测试用例文档

## 测试概述
本测试套件用于验证通知页面的功能完整性，包括基本展示、未读/已读标记、实时推送、响应式布局等功能。

## 测试环境
- 浏览器: Chromium, Firefox, WebKit (Playwright默认)
- 设备: 桌面端和移动端
- 网络条件: 正常网络连接

## 测试用例详情

### TC001: 通知页面基本功能测试
**目标**: 验证通知页面正确加载和显示
**前置条件**: 用户已登录系统
**步骤**:
1. 导航至 `/notifications` 页面
2. 验证页面标题包含"通知"
3. 验证页面标题为"通知中心"
4. 验证通知列表可见

**预期结果**:
- 页面正常加载
- 标题正确显示
- 通知列表元素存在且可见

### TC002: 未读/已读标记功能测试
**目标**: 验证通知的未读/已读状态标记功能
**前置条件**: 存在至少3条未读通知
**步骤**:
1. 验证初始状态下有3个未读通知
2. 点击第一个未读通知
3. 验证未读通知数量变为2，已读通知数量变为1
4. 点击"全部标记为已读"按钮
5. 验证所有通知变为已读状态

**预期结果**:
- 未读/已读状态正确切换
- 状态计数准确更新
- "全部标记为已读"功能正常

### TC003: WebSocket 实时推送测试
**目标**: 验证通过WebSocket接收实时通知
**前置条件**: WebSocket服务可用
**步骤**:
1. 检查WebSocket连接状态
2. 模拟接收新通知事件
3. 验证新通知是否在页面上显示
4. 验证通知属性（标题、内容、时间戳）

**预期结果**:
- WebSocket连接成功
- 实时通知正确显示
- 通知数据完整准确

### TC004: 响应式布局测试 - 桌面端
**目标**: 验证桌面端布局正确显示
**前置条件**: 页面在桌面分辨率下打开
**步骤**:
1. 设置视口大小为1280x720
2. 验证头部、侧边栏、主内容区域可见
3. 检查布局类名不包含移动端标识

**预期结果**:
- 所有桌面端布局元素正常显示
- 使用桌面端布局而非移动端布局

### TC005: 响应式布局测试 - 移动端
**目标**: 验证移动端布局正确显示
**前置条件**: 页面在移动设备分辨率下打开
**步骤**:
1. 设置视口大小为375x667
2. 验证移动端菜单按钮可见
3. 检查侧边栏适应移动端布局
4. 验证通知列表在移动端适配

**预期结果**:
- 移动端菜单按钮可见
- 布局适配移动端屏幕
- 通知列表正确显示

### TC006: 通知过滤功能测试
**目标**: 验证按类型过滤通知的功能
**前置条件**: 存在已读和未读通知
**步骤**:
1. 点击"全部"过滤器
2. 验证显示所有通知
3. 点击"未读"过滤器
4. 验证仅显示未读通知
5. 点击"已读"过滤器
6. 验证仅显示已读通知

**预期结果**:
- 过滤器功能正常工作
- 通知按类型正确过滤
- 通知计数正确更新

### TC007: 通知删除功能测试
**目标**: 验证删除通知功能
**前置条件**: 存在可删除的通知
**步骤**:
1. 记录初始通知数量
2. 点击第一个通知的删除按钮
3. 如有确认对话框则确认删除
4. 验证通知数量减少

**预期结果**:
- 通知被成功删除
- 通知列表数量正确更新

### TC008: 通知分页功能测试
**目标**: 验证通知列表分页功能
**前置条件**: 存在多页通知
**步骤**:
1. 验证当前页码显示为第1页
2. 点击下一页按钮
3. 验证页码更新为第2页
4. 点击上一页按钮
5. 验证页码返回第1页

**预期结果**:
- 分页导航正常工作
- 页码指示器正确更新
- 页面内容按分页显示

## 特殊测试场景

### 断网重连测试
- 模拟网络中断后恢复，验证WebSocket重新连接
- 验证离线期间的通知在重连后同步

### 大量通知测试
- 模拟大量通知（>100条）的显示性能
- 验证滚动和交互性能

### 多标签页测试
- 在多个标签页中打开通知页面
- 验证通知状态在各页面间同步

## 性能指标
- 页面加载时间 < 3秒
- 通知状态切换响应时间 < 500毫秒
- WebSocket消息接收延迟 < 1秒

## 兼容性测试
- Chromium 内核浏览器
- Firefox 浏览器
- Safari 浏览器
- 移动端 Chrome 和 Safari
