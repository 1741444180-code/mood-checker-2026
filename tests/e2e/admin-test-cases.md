# 后台管理测试用例文档

## 概述

本文档描述了后台管理系统的端到端测试用例，涵盖管理员登录、用户管理、内容审核、权限控制和数据统计等功能模块的测试。

## 测试环境

- 测试框架：Playwright
- 测试类型：端到端 (E2E)
- 测试环境：开发/测试环境
- 浏览器：Chrome (默认)

## 功能模块测试用例

### 1. 管理员登录测试

#### TC-ADMIN-001：成功登录后台管理系统
- **前置条件**：
  - 系统正常运行
  - 管理员账户存在且有效
- **测试步骤**：
  1. 访问 `/admin/login` 页面
  2. 输入有效的管理员用户名 (`admin`)
  3. 输入有效的管理员密码 (`admin123`)
  4. 点击登录按钮
- **预期结果**：
  - 成功跳转到管理面板 (`/admin/dashboard`)
  - 显示欢迎信息 "欢迎回来，管理员"
  - 侧边栏菜单可用

#### TC-ADMIN-002：使用错误凭据登录失败
- **前置条件**：
  - 系统正常运行
- **测试步骤**：
  1. 访问 `/admin/login` 页面
  2. 输入无效的用户名 (`invalid_user`)
  3. 输入错误的密码 (`wrong_password`)
  4. 点击登录按钮
- **预期结果**：
  - 显示错误提示 "用户名或密码错误"
  - 保持在登录页面

#### TC-ADMIN-003：登录表单验证
- **前置条件**：
  - 系统正常运行
- **测试步骤**：
  1. 访问 `/admin/login` 页面
  2. 不输入任何凭据
  3. 点击登录按钮
- **预期结果**：
  - 显示必填字段提示 "请输入用户名" 和 "请输入密码"

### 2. 用户管理功能测试

#### TC-USER-001：查看用户列表
- **前置条件**：
  - 管理员已登录
- **测试步骤**：
  1. 点击侧边栏 "用户管理" 菜单
  2. 等待页面加载完成
- **预期结果**：
  - 成功导航到 `/admin/users` 页面
  - 用户列表正确加载并显示
  - 至少显示一条用户记录

#### TC-USER-002：搜索用户
- **前置条件**：
  - 管理员已登录
  - 用户管理页面已打开
- **测试步骤**：
  1. 在用户搜索框中输入用户名 `testuser`
  2. 点击搜索按钮
- **预期结果**：
  - 显示匹配用户名的搜索结果
  - 结果中包含 `testuser` 用户

#### TC-USER-003：编辑用户信息
- **前置条件**：
  - 管理员已登录
  - 用户管理页面已打开
- **测试步骤**：
  1. 点击任意用户的 "编辑" 按钮
  2. 修改用户邮箱地址
  3. 点击保存按钮
- **预期结果**：
  - 显示成功消息 "用户信息更新成功"
  - 用户信息已更新到数据库

#### TC-USER-004：删除用户
- **前置条件**：
  - 管理员已登录
  - 用户管理页面已打开
- **测试步骤**：
  1. 记录当前用户总数
  2. 点击任意用户的 "删除" 按钮
  3. 在确认对话框中点击确认
- **预期结果**：
  - 用户从列表中移除
  - 当前用户总数比之前减少1

### 3. 内容审核功能测试

#### TC-CONTENT-001：查看待审核内容
- **前置条件**：
  - 管理员已登录
- **测试步骤**：
  1. 点击侧边栏 "内容审核" 菜单
  2. 等待页面加载完成
- **预期结果**：
  - 成功导航到 `/admin/content-review` 页面
  - 待审核内容列表正确加载
  - 显示至少一个待审核内容项

#### TC-CONTENT-002：通过内容审核
- **前置条件**：
  - 管理员已登录
  - 内容审核页面已打开
- **测试步骤**：
  1. 选择第一个待审核内容
  2. 点击 "通过" 按钮
  3. 在确认对话框中点击确认
- **预期结果**：
  - 显示成功消息 "内容已通过审核"
  - 该内容从待审核列表中移除

#### TC-CONTENT-003：拒绝内容审核
- **前置条件**：
  - 管理员已登录
  - 内容审核页面已打开
- **测试步骤**：
  1. 选择第一个待审核内容
  2. 点击 "拒绝" 按钮
  3. 在确认对话框中点击确认
- **预期结果**：
  - 显示成功消息 "内容已被拒绝"
  - 该内容从待审核列表中移除

#### TC-CONTENT-004：批量审核内容
- **前置条件**：
  - 管理员已登录
  - 内容审核页面已打开
- **测试步骤**：
  1. 选择多个待审核内容项
  2. 点击 "批量通过" 按钮
  3. 在确认对话框中点击确认
- **预期结果**：
  - 显示成功消息 "批量审核操作完成"
  - 所选内容从待审核列表中移除

### 4. 权限控制测试

#### TC-PERMISSION-001：管理员访问所有功能模块
- **前置条件**：
  - 管理员已登录
- **测试步骤**：
  1. 检查侧边栏菜单项
  2. 验证所有管理员功能是否可见
- **预期结果**：
  - "用户管理"、"内容审核"、"系统设置"、"数据统计" 等菜单项可见

#### TC-PERMISSION-002：普通用户无法访问管理员功能
- **前置条件**：
  - 普通用户已登录
- **测试步骤**：
  1. 登录普通用户账户
  2. 检查侧边栏菜单项
  3. 验证某些管理员功能是否隐藏
- **预期结果**：
  - "系统设置" 等高级功能菜单项不可见

#### TC-PERMISSION-003：权限分配功能
- **前置条件**：
  - 管理员已登录
  - 权限管理页面已打开
- **测试步骤**：
  1. 修改特定用户的权限设置
  2. 保存权限更改
- **预期结果**：
  - 显示成功消息 "权限已更新"
  - 用户权限已持久化到数据库

### 5. 数据统计准确性测试

#### TC-STATS-001：验证用户总数统计
- **前置条件**：
  - 管理员已登录
- **测试步骤**：
  1. 访问数据统计页面
  2. 记录仪表板上的用户总数
  3. 访问用户管理页面
  4. 统计实际用户数量
- **预期结果**：
  - 仪表板显示的用户总数与实际用户数量一致

#### TC-STATS-002：验证内容审核统计数据
- **前置条件**：
  - 管理员已登录
- **测试步骤**：
  1. 访问数据统计页面
  2. 记录待审核内容数量
  3. 访问内容审核页面
  4. 统计实际待审核内容数量
- **预期结果**：
  - 仪表板显示的待审核内容数与实际数量一致

#### TC-STATS-003：验证日活跃用户统计
- **前置条件**：
  - 管理员已登录
- **测试步骤**：
  1. 访问数据统计页面
  2. 检查今日活跃用户数显示
- **预期结果**：
  - 显示合理的日活跃用户数量（非负整数）

#### TC-STATS-004：验证统计图表加载
- **前置条件**：
  - 管理员已登录
- **测试步骤**：
  1. 访问数据统计页面
  2. 检查各类统计图表
- **预期结果**：
  - 用户增长图表、内容统计图表、收入图表等正确加载并显示

## 测试执行说明

1. 确保测试环境已准备好，包括数据库和服务器
2. 安装必要的依赖包 (`npm install` 或 `yarn install`)
3. 运行测试命令：
   ```bash
   # 运行所有后台管理测试
   npx playwright test tests/e2e/admin.spec.ts
   
   # 运行特定测试组
   npx playwright test --grep "管理员登录测试"
   
   # 运行测试并生成报告
   npx playwright test tests/e2e/admin.spec.ts --reporter=html
   ```

## 测试报告

- 执行完成后，Playwright 将生成详细的测试报告
- 包括测试通过/失败状态、执行时间、截图和视频（如配置）
- 报告路径：`playwright-report/index.html`

## 注意事项

1. 确保测试数据不会影响生产环境
2. 测试完成后清理测试数据（如需要）
3. 定期更新测试用例以适应功能变化
4. 在不同浏览器上验证测试兼容性
