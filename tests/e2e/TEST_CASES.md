# 消息和好友功能 E2E 测试用例文档

## 项目概述
本文档描述了消息中心和好友列表的端到端(E2E)测试用例，使用Playwright框架实现自动化测试。

## 测试范围
- 消息中心功能测试
- 好友管理功能测试
- 响应式布局测试

## 测试环境要求
- Playwright v1.40+
- Node.js 16+
- 支持的浏览器: Chromium, Firefox, WebKit
- 测试账户: test_user1, test_user2

## 消息中心测试用例 (messages.spec.ts)

### TC001: 消息发送功能测试
- **前置条件**: 用户已登录系统
- **测试步骤**:
  1. 导航到消息中心页面
  2. 点击发送新消息按钮
  3. 选择收件人
  4. 输入消息内容
  5. 点击发送按钮
- **预期结果**: 消息成功发送并在界面中显示发送状态

### TC002: 消息接收功能测试
- **前置条件**: 用户已登录系统，存在未读消息
- **测试步骤**:
  1. 导航到消息中心页面
  2. 检查未读消息指示器
  3. 验证消息列表内容
- **预期结果**: 正确显示接收到的消息

### TC003: 消息已读功能测试
- **前置条件**: 用户已登录系统，存在未读消息
- **测试步骤**:
  1. 导航到消息中心页面
  2. 点击未读消息
  3. 等待消息状态更新
- **预期结果**: 消息状态从"未读"变为"已读"

### TC004: 消息删除功能测试
- **前置条件**: 用户已登录系统，存在可删除的消息
- **测试步骤**:
  1. 导航到消息中心页面
  2. 选择要删除的消息
  3. 点击删除按钮
  4. 确认删除操作
- **预期结果**: 消息从列表中移除

### TC005: 消息中心响应式布局测试
- **前置条件**: 消息中心页面已加载
- **测试步骤**:
  1. 设置桌面分辨率 (1280x720)
  2. 设置平板分辨率 (768x1024)
  3. 设置移动分辨率 (375x667)
  4. 验证各分辨率下布局正确性
- **预期结果**: 在各种屏幕尺寸下，界面元素布局合理，功能可用

## 测试执行指令

运行所有测试：
```bash
npx playwright test
```

运行特定测试文件：
```bash
npx playwright test tests/e2e/messages.spec.ts
npx playwright test tests/e2e/friends.spec.ts
```

运行测试并查看UI：
```bash
npx playwright test --ui
```

运行测试并生成报告：
```bash
npx playwright test --reporter=html
```

## 测试数据准备
测试需要以下数据：
- 测试账户: test_user1 (主账户), test_user2 (用于交互测试)
- 预期好友关系: 初始状态下，两个测试账户不是好友
- 预期消息: 测试账户之间可能存在的历史消息

## 注意事项
1. 测试依赖于有效的登录凭据，请确保测试环境中存在测试账户
2. 某些功能（如实时消息）可能需要额外的等待时间来确保状态同步
3. 响应式测试覆盖主流设备尺寸，可根据实际需求调整
4. 在CI/CD环境中运行时，确保有足够的超时时间处理网络延迟

## 测试维护
- 当UI组件发生变化时，相应更新定位器选择器
- 当功能逻辑变更时，调整测试步骤和预期结果
- 定期检查测试数据的有效性，确保测试账户和状态正常
