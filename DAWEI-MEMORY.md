## 📞 沟通机制

### **定时汇报（每日）**

| 时间 | 汇报类型 | 汇报给谁 | 内容 |

| 19:30 | 启动汇报 | 黄金九 | 当日任务分配、预计完成时间 |

| 21:30 | 最终完成汇报 | 黄金九 | 100% 完成、交付物、质量评分 |

### **超时反查机制**

| 超时时长 | 行动 |
|---------|------|
| 5 分钟 | 黄金九@大伟询问原因 |
| 15 分钟 | 黄金九排查原因 |
| 30 分钟 | 黄金九直接介入处理 |
| 60 分钟 | 黄金九向建权汇报，建议更换负责人 |

### **连续性保障机制**

**每完成一个 Day：**
1. 提交代码到 Git
2. 创建进度记忆文件（PROGRESS/DayX-完成报告.md）
3. 验证文件完整性
4. 填写 Git 提交记录

**重启会话前：**
1. 确认所有文件已提交（git status 干净）
2. 更新进度记忆文件
3. 创建/更新重启说明（RESTART-INSTRUCTIONS.md）
4. 更新大伟记忆文件（本文件）

**重启会话后：**
1. 读取进度记忆文件（PROGRESS/DayX-完成报告.md）
2. 读取大伟记忆文件（DAWEI-MEMORY.md）
3. 验证 Git 提交记录
4. 验证文件完整性
5. 确认下一步任务
6. 继续工作
