# 消息通知页面设计规范

## 📋 概述

本文档定义了消息通知页面的设计规范，包含 PC 端和移动端两种布局。设计遵循现代、简洁、易用的原则，确保用户能够高效地管理和查看通知消息。

- **卡片圆角**: `12px`
- **头像尺寸**: `40px × 40px`
- **安全区域**: 适配 `safe-area-inset-bottom`

## 🧩 组件规范

### 1. 通知列表项

#### 结构
```
┌─────────────────────────────────────────────────────┐
│ [头像]  标题                                         │
│         时间 | 类型                           [操作]  │
│         消息内容（最多 2 行，超出省略）                │
└─────────────────────────────────────────────────────┘
```

#### 状态
- **默认**: 白色背景 `#ffffff`
- **未读**: 浅蓝背景 `#e8f3ff` + 左侧蓝色边框（移动端）
- **悬停**: 浅灰背景 `#f5f6f7`
- **点击**: 加深悬停色

#### 交互
- 点击卡片：打开消息详情
- 点击「标记已读」按钮：将消息标记为已读
- 点击「删除」按钮：删除消息（可二次确认）

### 2. 筛选标签页

#### PC 端
- 水平排列，带下划线指示器
- 激活状态：蓝色文字 `#3370ff` + 蓝色下划线
- 默认状态：灰色文字 `#646a73`

#### 移动端
- 横向滚动，胶囊样式
- 激活状态：蓝色背景 `#3370ff` + 白色文字
- 默认状态：浅灰背景 `#f5f6f7` + 灰色文字

### 3. 推送通知弹窗（PC 端）

#### 位置
- 固定在页面右上角
- 距离顶部：`24px`
- 距离右侧：`24px`
- 宽度：`400px`

#### 动画
- 入场：从右侧滑入 `slideIn`
- 持续时间：`0.3s`
- 缓动函数：`ease-out`

#### 结构
```
┌──────────────────────────────────────┐
│ [图标]  标题                    [×]   │
│         消息内容                     │
│         [查看] [稍后]                │
└──────────────────────────────────────┘
```

### 4. 推送通知横幅（移动端）

#### 位置
- 固定在页面顶部
- 距离顶部：`12px`
- 左右边距：`12px`

#### 动画
- 入场：从上方下滑 `slideDown`
- 持续时间：`0.3s`
- 缓动函数：`ease-out`

### 5. 操作按钮

#### 主要按钮
- 背景：`#3370ff`
- 文字：`#ffffff`
- 圆角：`6px` (PC) / `8px` (移动)
- 内边距：`8px 16px`

#### 次要按钮
- 背景：`#f5f6f7`
- 文字：`#646a73`
- 边框：`1px solid #e7e8ea`

#### 危险按钮
- 背景：`#f53f3f`
- 文字：`#ffffff`

#### 图标按钮
- 尺寸：`32px × 32px` (PC) / `36px × 36px` (移动)
- 圆角：`6px` / `8px`
- 悬停：蓝色边框 + 蓝色图标

## ♿ 无障碍设计

### 键盘导航
- `Tab`: 在可交互元素间切换
- `Enter`: 激活当前焦点元素
- `Escape`: 关闭弹窗/横幅

### 屏幕阅读器
- 所有图标按钮添加 `title` 属性
- 未读状态使用语义化标记
- 时间信息使用 `<time>` 标签

### 对比度
- 主要文字对比度：`≥ 4.5:1`
- 大文字对比度：`≥ 3:1`

---

## 🎭 动效规范

### 过渡动画
- 默认过渡：`all 0.2s`
- 入场动画：`0.3s ease-out`
- 悬停反馈：`0.2s`

### 动画类型
| 动画 | 用途 | 时长 |
|------|------|------|
| `slideIn` | PC 推送弹窗入场 | 0.3s |
| `slideDown` | 移动推送横幅入场 | 0.3s |
| `slideUp` | 底部弹_sheet 入场 | 0.3s |

## 🔧 开发建议

### 性能优化
1. 列表项使用虚拟滚动（大量数据时）
2. 图片使用懒加载
3. 动画使用 `transform` 和 `opacity` 属性

### 兼容性
- 现代浏览器（Chrome 80+, Safari 13+, Firefox 75+）
- 移动端 iOS 13+, Android 8+

### 可访问性
- 确保所有交互元素可键盘访问
- 提供足够的点击区域（移动端 ≥ 44px）
- 使用语义化 HTML 标签
