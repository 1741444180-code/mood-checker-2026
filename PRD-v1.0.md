## 一、项目概述

### 1.1 项目背景

心情打卡网站是一个帮助用户记录每日心情、追踪情绪变化、提供数据统计的 Web 应用。用户可以通过选择表情包快速打卡，查看心情日历和统计图表，支持 PC 端和手机端响应式设计。

### 1.2 项目目标

- ✅ 帮助用户记录每日心情
- ✅ 提供心情日历和统计图表
- ✅ 支持多用户注册登录
- ✅ 适配 PC 端和手机端
- ✅ 快速上线验证（MVP 版本）

### 1.3 参考界面

**已提供 4 张参考图：**
1. 移动端打卡界面（5 种心情表情包）
2. PC 端打卡界面（居中布局）
3. 心情日历界面（日历视图，颜色表示心情）
4. 心情统计界面（饼图 + 柱状图）

**界面风格：**
- 简洁、小清新
- 蓝白色调为主
- 圆角卡片设计
- 数据可视化清晰

## 三、技术架构

### 3.1 技术栈推荐

#### 方案 A：现代全栈（推荐）
| 层级 | 技术选型 | 理由 |
|-----|---------|------|
| 前端框架 | **Next.js 14** + React | SSR/SSG 支持，SEO 友好，生态成熟 |
| UI 组件库 | **shadcn/ui** + Tailwind CSS | 简洁风格，易定制，蓝白主题适配 |
| 状态管理 | Zustand | 轻量级，比 Redux 简单 |
| 后端框架 | Node.js + Express 或 Next.js API Routes | 前后端同语言，开发效率高 |
| 数据库 | **PostgreSQL** + Prisma ORM | 关系型数据，类型安全 |
| 认证系统 | **NextAuth.js** | 支持多种登录方式，开箱即用 |
| 部署平台 | **Vercel**（前端）+ **Railway**（数据库） | 免费额度够用，自动 CI/CD |
| 图表库 | **Recharts** | React 友好，轻量 |

#### 方案 B：快速原型（更简单）
| 层级 | 技术选型 | 理由 |
|-----|---------|------|
| 前端 | **Vue 3** + Vite | 上手快，中文文档完善 |
| UI 组件 | **Element Plus** | 组件丰富，主题定制方便 |
| 后端 | **Python + FastAPI** | 开发速度快，自动 API 文档 |
| 数据库 | **SQLite**（初期）→ PostgreSQL | 零配置，后期可迁移 |
| 部署 | **Docker** + 任意云服务器 | 一键部署 |

### 3.2 数据库设计

```sql
-- 用户表
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  username VARCHAR(50) UNIQUE NOT NULL,
  email VARCHAR(100) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  avatar VARCHAR(255),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  last_login TIMESTAMP,
  timezone VARCHAR(50) DEFAULT 'UTC'
);

-- 心情记录表
CREATE TABLE mood_records (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id),
  date DATE NOT NULL,
  mood_level INTEGER CHECK (mood_level >= 1 AND mood_level <= 5),
  mood_type VARCHAR(50) NOT NULL,
  note TEXT,
  tags VARCHAR(100),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, date)
);

-- 心情类型表
CREATE TABLE mood_types (
  id SERIAL PRIMARY KEY,
  name VARCHAR(50) NOT NULL,
  color VARCHAR(7) NOT NULL,
  icon VARCHAR(50) NOT NULL,
  sort_order INTEGER DEFAULT 0
);

-- 用户设置表
CREATE TABLE user_settings (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id),
  reminder_time TIME,
  reminder_enabled BOOLEAN DEFAULT false,
  weekly_report BOOLEAN DEFAULT false,
  public_profile BOOLEAN DEFAULT false,
  theme VARCHAR(50) DEFAULT 'light'
);

-- 统计数据表 (缓存)
CREATE TABLE statistics (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id),
  period VARCHAR(20) NOT NULL,
  mood_distribution JSONB,
  trend_data JSONB,
  generated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
