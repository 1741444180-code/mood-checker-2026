|---------|------|

### **连续性保障机制**

**每完成一个 Day：**
1. 提交代码到 Git
2. 创建进度记忆文件（PROGRESS/DayX-完成报告.md）
3. 验证文件完整性
4. 填写 Git 提交记录

**重启会话前：**
1. 确认所有文件已提交（git status 干净）
2. 更新进度记忆文件
3. 创建/更新重启说明（RESTART-INSTRUCTIONS.md）

**重启会话后：**
1. 读取进度记忆文件（PROGRESS/DayX-完成报告.md）
2. 验证 Git 提交记录
3. 验证文件完整性
4. 确认下一步任务
5. 继续工作
